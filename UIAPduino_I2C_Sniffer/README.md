# UIAPduino I2C Sniffer

## 概要 (Overview)
CH32V003 (UIAPduino) の限られたリソース(RAM 2KB)を極限まで引き出した、PC不要のスタンドアローンI2Cロジックアナライザーです。
「フリスクケースに収まる最強のスニッファー」をコンセプトに、ポケットからサッと取り出して直感的にI2C通信を解析することを目的として設計されました。

## 特徴 (Features)
* **スタンドアローン駆動:** PCや外部アプリの接続は一切不要。OLEDディスプレイとマイコン単体で完結します。
* **波形同期オーバーレイ:** `SCL` / `SDA` 波形の直下（データ確定タイミング）に、デコードされた16進数データ(`0x??`)が寄り添って表示されます。
* **スマートリフレッシュ:** スクロール操作を行った時のみ画面を更新し、画面のフリッカー（チラつき）を排除した静止画のような視認性を誇ります。
* **メモリ最適化記録:** イベントベースの記録方式によりRAM枯渇を回避し、ハードウェア限界の最大250サンプルまで高精度にキャプチャ可能です。
* **デュアルログ表示:** 波形直下の純粋なペイロード（`0x??`）表示に加え、画面下部にはプロトコルの成否（ACK/NACK）を含めた通信全体のログ一覧が表示されます。

## ピン配置 (Pin Map)
CH32V003 (UIAPduino) の以下のピンを使用します。

* `PA1` : 読込中LED (Active)
* `PA2` : エラーLED (Error / NACK検出時に点灯)
* `PC1` : OLED SDA (ディスプレイ通信用)
* `PC2` : OLED SCL (ディスプレイ通信用)
* `PC3` : `[LEFT]` ボタン (波形の左スクロール)
* `PC4` : `[SEL]` ボタン (録画開始 / 閲覧終了)
* `PC5` : `[RIGHT]` ボタン (波形の右スクロール)
* `PC6` : ターゲット SDA (測定対象のSDAピンへ接続)
* `PC7` : ターゲット SCL (測定対象のSCLピンへ接続)

## 使い方 (How to Use)
1. **待機モード (STANDBY MODE):** 電源を投入すると `STANDBY MODE` と表示され、待機状態になります。
2. **キャプチャ開始:** 測定対象の機器と `PC6`, `PC7`, および `GND` を接続し、`[SEL]` ボタンを押します。画面が `WATCHING...` に切り替わり、I2Cのスタートコンディションをトリガーとして記録を開始します。
3. **波形・ログの解析:** 記録が完了すると解析画面が表示されます。
   * `[LEFT]` / `[RIGHT]` ボタンを押して、波形を左右にスクロールします。
   * 画面下部で通信の流れを確認しつつ、上部の波形と同期データで正確なタイミングとデータ値を確認してください。
4. **終了:** 解析画面で `[SEL]` を長押しすると、待機モードに戻り次のキャプチャが可能になります。

## 画面の見方 (Display Guide)
* **Page 0-1:** `SCL` および `SDA` の物理波形。
* **Page 2:** 波形同期データ (`0x78` などの16進数、`S` = Start, `P` = Stop)。
* **Page 4:** ステータス表示 (`E:` = 総キャプチャサンプル数, `P:` = 現在のスクロールオフセット位置)。
* **Page 6-7:** プロトコル全体ログ (例: `S 78A 00A P` ※ A = ACK, N = NACK)。
