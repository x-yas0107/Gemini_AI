【プロジェクト概要】
本プロジェクトは、リソースが極めて限られたマイコン（最大16KB ROM / 2KB RAM）で動作するコンソール型ハッキングOSです。重い標準ライブラリを一切捨て、マイコンのレジスタを直接叩く（ベアメタルプログラミング）超軽量設計により、OSライクな機能をフラッシュメモリ容量の90%（約14.8KB）に実装しました。

【基本操作コマンド】 [cite: 1]
  S          : I2Cバススキャン。接続中のデバイスIDを探し、最後にみつけたIDを記憶。 [cite: 1]
  R / RUN    : 自動実行。Sで見つけたID専用のアドレス(13xxx番地)を計算して実行。 [cite: 1]
  L          : 辞書リスト。メモリに登録済みのデバイス名一覧を表示。 [cite: 1]
  C          : OLEDクリア。液晶表示を真っ白に消去。 [cite: 1]
  H / ?      : ヘルプメニューを表示。 [cite: 2]

【メモリ読み書きコマンド】 [cite: 2]
  B [Addr]   : バルク転送(推奨)。 [cite: 2]
               例: "B 13C00" -> その後 "011078100010A702204F204BFFX" を貼り付け。 [cite: 2]
               ※最後に 'X' を入れることで書き込み確定。上書きに最適。 [cite: 2]
  D [A] [L]  : データ確認(Dump)。指定アドレス(A)から指定数(L)を表示。 [cite: 2]
               例: "D 13C00 16" [cite: 2]
  E [Addr]   : セクタ消去。指定アドレスを含む4KBの範囲をFF(空)にする。 [cite: 2]
               例: "E 13000" [cite: 2]
  P [A] [D..]: 手動書き込み。小規模な修正用。 [cite: 3]
               例: "P 100 3C 4F 4B" (アドレス100に 3C, 4F, 4B を書く) [cite: 3]

【高度なコマンド】 [cite: 3]
  X [Addr]   : VM直接実行。アドレスを直接指定してスクリプトを走らせる。 [cite: 3]
               例: "X 13C00" [cite: 3]
  W [A][W][D]: 動画再生。アドレス(A)から幅(W)ドット分、速度(D)msで描画。 [cite: 3]
               例: "W 20000 500 10" (Bad Apple!!等の再生用) [cite: 3]
  F 1234     : システム初期化。メモリを消去し専用設定でフォーマット。 [cite: 3]
  M [X][P][A]: 8x8画像描画。メモリ(A)の8バイトを、位置(X)ページ(P)に表示。 [cite: 3]
  T [I]      : [WIP] Misakiフォント表示。指定インデックス(I)の文字を描画（※SOS参照）
  K          : ベーステスト。OLEDの基本描画テストを実行。

【🆘 開発者からのSOS / TODO (Help Wanted!) 🆘】
--------------------------------------------------------------------------------
現在、「Tコマンド（Misakiフォント表示）」のOLED描画ロジックが未完成です。
外部Flashメモリから指定したインデックスの文字データは正しく読み出せていますが、
美咲フォントの縦横データ構造と、SSD1306(OLED)のページアドレッシングの仕様が噛み合わず、
OLEDに描画すると文字が「横線」に化けてしまうバグが存在します。

マイコンの残りROM容量は約10%（1.5KB）です。
この制約の中で描画エンジンを改修し、漢字を正しく表示させてくれる
コントリビューターを全力で募集しています！Pull Requestをお待ちしています！

【VM指示書(スクリプト) 命令コード表】 [cite: 3]
--------------------------------------------------------------------------------
  01      : I2C開始 (Start) [cite: 3]
  02      : I2C停止 (Stop) [cite: 4]
  10 [XX] : 1バイト送信 (XXを送る) [cite: 4]
  11      : 1バイト受信 (ACK継続) [cite: 4]
  12      : 1バイト受信 (NACK終了) [cite: 4]
  20 [XX] : ターミナルに文字表示 (XXを表示) [cite: 4]
  21      : ターミナルに直前の受信値を16進数で表示 [cite: 4]
  30 [XX] : XXミリ秒待機 (Delay) [cite: 4]
  FF      : スクリプト終了 (End) [cite: 4]

--------------------------------------------------------------------------------
【実践例：OLED反転テスト】 [cite: 4]
--------------------------------------------------------------------------------
  1. 「B 13C00」を入力。 [cite: 4]
  2. 「011078100010A702204F204BFFX」を貼り付ける。 [cite: 4]
  3. 「S」を入力してデバイス(0x3C)を認識させる。 [cite: 4]
  4. 「R」または「RUN」を入力して実行！ [cite: 4]
